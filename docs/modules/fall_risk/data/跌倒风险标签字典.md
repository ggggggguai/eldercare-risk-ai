# 跌倒风险标签字典

本文是跌倒风险方向的统一标签字典。所有人工标注、自动导入标注和训练数据转换都必须使用这里的标签；标注员不得临时发明新标签。

## 1. 使用原则

- 标注员只记录视频里能看见的事实，不做医学诊断，不判断老人真实健康状况。
- 每个动作片段优先选择一个最主要的动作标签；确实同时发生时，在备注中补充次要现象。
- 看不清、被遮挡、人物离开画面或无法确认对象时，用 `U01`，不要猜。
- 风险等级标注由经过培训的复核人员或项目负责人确认，不作为第一阶段人工动作标注任务。
- 同一套标签字典服务所有数据集，但每个数据集只使用自己能支持的标签子集。
- 自动转换结果只能写入 `data/annotations/fall_risk/generated/v1/` 的来源专属目录，并保持 `eligibility=false`；不能因格式校验通过就提升为根目录正式标签。
- 人工序数标签只写 `risk_level`，禁止填写模型预测字段 `risk_score`。

## 2. 动作级标签

动作级统一契约位于：

```text
data/annotations/fall_risk/action_labels.jsonl
```

CVAT 转换候选先写入 `generated/v1/cvat_<export-id>/action_labels.jsonl`。当前 schema 的必填字段为：

```text
label_id                  source_record_id
source_annotation_path    source_annotation_sha256
source_export_id           asset_id
video_id                   file_path
subject_id                 scene
view                       action_id
action_name                event_type
start_time                 end_time
start_frame                end_frame
frame_index_base           labeler
review_status              eligibility
review_evidence_ids        quality
note                       source
cvat_task_id               cvat_track_id
bbox_start                 bbox_end
```

不允许额外字段。转换器生成的 `label_id`、来源 ID 和来源 SHA-256 用于追溯；时间由 manifest 中该视频的 `fps_num/fps_den` 换算，统一帧号使用 `frame_index_base=0`。初始候选固定为 `review_status=pending`、`eligibility=false`、`review_evidence_ids=[]` 和 `source=cvat`。

### 2.1 A 类：正常动作

| 编号    | 英文名            | 中文名  | 什么时候用                       | 不要用于               |
| ----- | -------------- | ---- | --------------------------- | ------------------ |
| `A01` | `normal_walk`  | 正常行走 | 人在连续走路，身体基本稳定，无明显踉跄、扶物或异常下沉 | 只是站立、转身、坐下；明显拖步或摇晃 |
| `A02` | `normal_turn`  | 正常转身 | 人完成转身，过程稳定，没有明显停顿、晃动或扶物     | 转身时明显失衡或需要扶物       |
| `A03` | `normal_sit`   | 正常坐下 | 从站立到坐下，动作连续，最终坐稳            | 身体突然下坠、坐下明显失控      |
| `A04` | `normal_stand` | 正常起身 | 从坐姿到站立，能一次站起并站稳             | 起身很慢、多次尝试或起身失败     |

### 2.2 B 类：轻到中度风险动作

| 编号    | 英文名                 | 中文名  | 什么时候用                  | 不要用于                |
| ----- | ------------------- | ---- | ---------------------- | ------------------- |
| `B01` | `slow_walk`         | 缓慢行走 | 明显走得慢，但仍稳定，没有拖脚、扶物或失衡  | 正常速度行走；已出现拖步、小碎步或摇晃 |
| `B02` | `dragging_walk`     | 拖步   | 脚抬不起来，脚尖或脚底像拖着地面移动     | 只是走得慢但脚能正常抬起        |
| `B03` | `shuffling_walk`    | 小碎步  | 步幅很小、步频偏密、移动缓慢         | 正常小步调整姿势或短距离挪动      |
| `B04` | `swaying_walk`      | 行走摇晃 | 走路时身体左右晃，稳定性差，但未明显差点摔倒 | 明显踉跄后恢复，应标 `C03`    |
| `B05` | `unstable_turn`     | 转身不稳 | 转身时明显晃动、停顿、重心不稳        | 正常稳定转身              |
| `B06` | `slow_sit_to_stand` | 起身缓慢 | 能站起来，但动作慢、费力、需要较长时间稳定  | 多次起身失败，应标 `C01`     |

### 2.3 C 类：高风险前兆动作

| 编号    | 英文名                        | 中文名       | 什么时候用                         | 不要用于                              |
| ----- | -------------------------- | --------- | ----------------------------- | --------------------------------- |
| `C01` | `failed_sit_to_stand`      | 起身失败      | 尝试站起但没有成功，或多次尝试后才站起           | 只是起身慢但一次成功，应标 `B06`               |
| `C02` | `wall_support_walk`        | 扶墙/扶物行走   | 行走或移动过程中持续扶墙、桌、椅、扶手等保持平衡      | 坐下或起身时短暂接触家具，需结合 `A03/A04/B06` 备注 |
| `C03` | `stumble_recovery`         | 踉跄后恢复     | 身体明显失衡、脚步错乱或差点摔倒，但最后站稳        | 真正倒地，应标 D 类跌倒                     |
| `C04` | `rapid_support_contact`    | 快速扶物      | 快要失衡时突然扶墙、扶桌、扶椅子恢复平衡          | 正常扶椅坐下或日常拿取物品                     |
| `C05` | `rapid_body_drop_recovery` | 身体快速下沉后恢复 | 身体突然往下掉、膝髋快速弯曲或重心快速下降，但未倒地并恢复 | 正常坐下、下蹲或弯腰拾物                      |

### 2.4 D 类：跌倒和跌倒后状态

| 编号    | 英文名                      | 中文名   | 什么时候用                     | 不要用于        |
| ----- | ------------------------ | ----- | ------------------------- | ----------- |
| `D01` | `forward_fall`           | 向前跌倒  | 身体明显向前倒下，失去支撑并接触地面、床边、椅旁等 | 主动趴下或正常弯腰   |
| `D02` | `lateral_fall`           | 侧向跌倒  | 身体明显向左或向右倒下，失去支撑          | 只是侧身坐下或侧向跨步 |
| `D03` | `backward_fall`          | 向后跌倒  | 身体明显向后倒下，失去支撑             | 正常坐回椅子或床上   |
| `D04` | `long_static_after_fall` | 跌倒后静止 | 跌倒后躺、趴或侧卧在地面等位置，持续不动或无法起身 | 睡觉、休息、主动躺下  |

### 2.5 U 类：无法判断

| 编号    | 英文名               | 中文名  | 什么时候用                           | 备注要求                              |
| ----- | ----------------- | ---- | ------------------------------- | --------------------------------- |
| `U01` | `unable_to_judge` | 无法判断 | 画面太暗、严重遮挡、人物太小、多人对象无法确认、动作只出现一半 | 必须写明原因，例如“遮挡严重”“人物离开画面”“无法确认是否倒地” |

## 3. 事件级标签

事件级统一契约位于：

```text
data/annotations/fall_risk/event_labels.jsonl
```

事件级标签用于近跌倒、跌倒、长时间静止等时间窗口。CVAT 映射事件和 LE2I 官方事件必须分来源保存为候选，人工确认前不得互相覆盖或写入根目录。

两类事件共用以下必填字段：

```text
label_id                  source_record_id
source_annotation_path    source_annotation_sha256
asset_id                  video_id
event_type                start_time
end_time                  start_frame
end_frame                 frame_index_base
severity                  label_source
review_status             eligibility
review_evidence_ids       note
```

`label_source=cvat_action_mapping` 还必须包含：

```text
source_export_id          source_action_id
source_action_name        source_action_label_id
mapping_version           cvat_task_id
cvat_track_id
```

映射事件与父动作的来源、资产、视频、边界、事件类型和 CVAT track 必须一致，初始状态为 `pending/false/[]`。

`label_source=le2i_txt` 还必须包含：

```text
source_start_frame        source_end_frame
source_frame_index_base
```

LE2I TXT 的源帧从 1 开始，统一帧从 0 开始；初始状态为 `review_status=auto_imported`、`eligibility=false`、`review_evidence_ids=[]`。TXT 的 `0/0` 表示显式无跌倒窗口，不生成伪事件；`Lecture room` 和 `Office` 无 TXT，不得生成官方事件。

| 事件类型                   | 中文名   | 对应动作证据        | 说明             |
| ---------------------- | ----- | ------------- | -------------- |
| `normal_activity`      | 正常活动  | `A01-A04`     | 正常行走、坐下、起身、转身等 |
| `gait_instability`     | 步态不稳  | `B01-B04`     | 慢走、拖步、小碎步或行走摇晃 |
| `near_fall`            | 近跌倒   | `C03`         | 明显踉跄但最终恢复稳定     |
| `sit_stand_difficulty` | 坐站困难  | `B06/C01`     | 起身缓慢、起身失败或多次尝试 |
| `unstable_turning`     | 转身不稳  | `B05`         | 转身时出现重心不稳      |
| `wall_support`         | 扶墙/扶物 | `C02/C04`     | 持续扶物行走或突然扶物恢复  |
| `rapid_body_drop`      | 快速下沉  | `C05`         | 身体快速下降但未形成跌倒   |
| `fall`                 | 跌倒    | `D01/D02/D03` | 明确失去支撑并倒下      |
| `long_static`          | 长时间静止 | `D04`         | 跌倒后持续不动或无法起身   |
| `uncertain`            | 不确定事件 | `U01`         | 事件存在可能，但证据不足   |

校验器还保留 `sudden_stop_recovery` 和 `recovery` 枚举供后续恢复事件契约使用，但当前 CVAT/LE2I 导入器不会生成它们；没有可追溯来源时不得手工伪造。

## 4. 风险级标签

风险级标签统一契约位于：

```text
data/annotations/fall_risk/risk_labels.jsonl
```

风险级标签用于 `functional_proxy` 或 `longitudinal_baseline` 任务，不在第一轮动作标注中直接填写。没有真实人工或临床 proxy 结果时，该 JSONL 必须保持零条记录。

当前 schema 的必填字段为：

```text
label_id                  asset_id
task_type                 subject_id
start_time                end_time
risk_level                risk_factors
label_source              review_status
eligibility               review_evidence_ids
```

可选字段只有 `video_id`、`review_record_ids` 和 `note`。非视频 GSTRIDE/LTMM 表格使用 `asset_id`，不要求 `video_id`。约束如下：

- `task_type` 只能是 `functional_proxy` 或 `longitudinal_baseline`。
- `label_source` 只能是 `manual_consensus` 或 `clinical_proxy`。
- `risk_level` 是 0-4 的人工序数真值；人工标签禁止 `risk_score`，连续分数只能出现在模型预测。
- `eligibility=true` 时，`review_evidence_ids` 必须精确列出当前有效的独立复核记录，并满足 manifest、许可和正式状态门槛。

| 等级  | 中文名  | 可观察证据                  | 建议动作编码            |
| ---:| ---- | ---------------------- | ----------------- |
| `0` | 正常   | 正常行走、正常坐站、无异常停留        | `record_only`     |
| `1` | 轻微风险 | 轻微慢走、短暂停顿、轻微摇晃，但无明显失衡  | `observe`         |
| `2` | 中度风险 | 拖步、小碎步、明显摇晃、起身缓慢、转身不稳  | `remind_user`     |
| `3` | 高风险  | 近跌倒、扶物恢复、起身失败、明显失衡但未倒地 | `notify_guardian` |
| `4` | 紧急风险 | 明确跌倒、跌倒后长时间静止、疑似无法起身   | `emergency_alert` |

## 5. 复核记录 schema

复核记录写入 `data/annotations/fall_risk/annotation_review_log.jsonl`。没有真实人工复核时文件必须为空，不能放示例记录。每条记录必填：

```text
review_id                 label_id
label_type                reviewer_id
decision                  reviewed_at
reason_code               note
```

可选字段为：

```text
previous_record_sha256    result_record_sha256
supersedes_review_id
```

字段约束：

- `label_type` 只能是 `action`、`event` 或 `risk`；`decision` 只能是 `approve`、`revise`、`reject`、`conflict` 或 `adjudicate`。
- `reviewed_at` 必须是带时区的 ISO 8601 时间；`reviewer_id` 必须是脱敏标识，不能写姓名、邮箱或手机号。
- `approve` 和 `adjudicate` 必须包含 `result_record_sha256`，且它必须等于最终标签整条记录的规范化 SHA-256。规范化方式是按 key 排序、UTF-8、无多余空白的 JSON，即 `json.dumps(record, ensure_ascii=False, sort_keys=True, separators=(",", ":"))` 后计算 SHA-256。
- 任何还会被后继 review 取代的记录也必须提供 `result_record_sha256`；因此待仲裁的 `conflict` 要记录其争议结果哈希，供后继 `previous_record_sha256` 绑定。
- 同一条 `eligibility=true` 标签至少需要两个不同 `reviewer_id` 的当前有效 `approve/adjudicate`；标签中的 `review_evidence_ids` 必须与这些有效 review ID 精确一致。两个 review ID 来自同一 reviewer 不算双人复核。
- 出现冲突时，后继记录用 `supersedes_review_id` 指向前一记录，并令 `previous_record_sha256` 等于前一记录的 `result_record_sha256`。`conflict` 必须由直接后继的 `adjudicate` 解决；仲裁人必须不同于整条前置复核链中的所有 reviewer，时间必须更晚。完成仲裁后仍需另一名独立 reviewer 的有效批准，才能达到双人门槛。
- 未解决的 `revise/reject/conflict` 是 blocker；被后继记录取代的 review 不再作为有效证据。

复核哈希绑定的是已经写好最终 `review_evidence_ids` 和 `eligibility` 的完整标签记录。任何字段变化都会改变哈希，必须重新复核，不能只修改 review log 来绕过校验。

## 6. 人员画像 schema

`data/annotations/fall_risk/subject_profiles.json` 是单个 JSON 对象，不是 JSONL。空模板固定为：

```json
{
  "schema_version": "fall-risk-subject-profiles-v1",
  "subjects": []
}
```

每个 `subjects` 元素必填 `subject_id`、`profile_version`、`profile_source`、`review_status`、`consent_id` 和 `features`，可选 `note`。`subject_id` 必须是脱敏且不能为 `unknown`；`features` 必须是 JSON 对象，不得包含联系方式。正式模式要求画像为 `reviewed/final` 且 `consent_id` 非空。没有真实画像、同意记录或特征时保留空数组，不得生成占位人员。

## 7. 质量和备注标记

质量标记不替代动作标签。它们用于解释为什么某段标注可信或不可信。

| 标记                       | 中文名    | 什么时候用              |
| ------------------------ | ------ | ------------------ |
| `clear`                  | 画面清楚   | 人体主体清晰可见，动作边界明确    |
| `partial_occlusion`      | 部分遮挡   | 身体局部被遮挡，但主要动作仍可判断  |
| `heavy_occlusion`        | 严重遮挡   | 关键动作被挡住，通常配合 `U01` |
| `low_light`              | 光线不足   | 画面过暗导致判断困难         |
| `off_screen`             | 离开画面   | 人物部分或完全离开画面        |
| `multi_person_uncertain` | 多人对象不清 | 无法确认应该标哪一个人        |

## 8. 时间边界规则

| 场景    | 开始时间                 | 结束时间                |
| ----- | -------------------- | ------------------- |
| 行走    | 第一步明显开始移动            | 停止行走或切换到转身、坐下、跌倒等动作 |
| 坐下    | 身体明显开始向下坐            | 坐稳、身体不再继续下降         |
| 起身    | 身体明显离开座位或开始上升        | 站稳、身体不再明显晃动         |
| 近跌倒   | 失衡、急停、快速下沉或突然扶物开始    | 恢复稳定，或转为真正跌倒        |
| 跌倒    | 身体开始失去支撑并向地面/床边/椅旁倒下 | 身体接触并稳定在倒地/倒卧状态     |
| 跌倒后静止 | 跌倒动作结束后开始静止          | 明显起身、移动或视频结束        |

时间可以先精确到 `0.1s`；如果工具只能按帧标注，则同时保存 `start_frame/end_frame`。

## 9. 常见混淆

| 容易混淆           | 正确判断                             |
| -------------- | -------------------------------- |
| 走得慢 vs 跌倒风险高   | 标注员只标 `B01`，不要直接写高风险             |
| 慢慢坐下 vs 身体快速下沉 | 正常可控坐下标 `A03`；突然下沉后恢复标 `C05`     |
| 近跌倒 vs 跌倒      | 最后站住了是近跌倒；明确倒下并失去支撑是跌倒           |
| 正常扶椅坐下 vs 快速扶物 | 日常坐下借扶不一定是 `C04`；突然失衡后扶物才是 `C04` |
| 画面看不清 vs 没有事件  | 看不清标 `U01`，不能当作正常负样本             |
