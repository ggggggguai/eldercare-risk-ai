# 公开跌倒事件数据集使用说明

本文说明当前已下载并解压的公开跌倒/步态数据集如何用于本项目的跌倒风险算法研发。重点是边界：它们适合验证跌倒事件检测链路或步态链路，但不能证明真实老人长期跌倒风险预警已经达标。

## 本地数据状态

| 数据集 | 本地路径 | 下载状态 | 主要用途 |
|---|---|---|---|
| LE2I/IMViA Fall Detection Dataset | `data/external/le2i_imvia/raw/FallDataset/` | 已解压完整 AVI；部分子集有 txt 标注 | 固定烟测、跌倒时间窗验证、检测/姿态/近跌倒链路调试 |
| Video-Based Fall Detection Dataset with 2017 Activities | `data/external/fall_detection_2017/raw/VideoDataset/` | 已下载并解压完整 zip | 主事件数据集，验证视频跌倒/ADL 区分 |
| UR Fall Detection Dataset | `data/external/ur_fall/raw/` | MP4/CSV 核心子集完整；图片序列 zip 部分保留且未展开 | 小规模交叉验证、快速烟测、RGB-D/IMU 后续扩展 |

## 2. Figshare 2017 Activities 数据集

来源：

```text
https://figshare.com/articles/dataset/Sensor-Based_Fall_Detection_Dataset_with_2017_Activities_from_29_Subjects/28596332
```

本地文件：

```text
data/external/fall_detection_2017/raw/VideoDataset.zip
data/external/fall_detection_2017/raw/VideoDataset/
```

许可：

```text
CC BY 4.0
```

推荐用途：

- 作为当前替代 IPML 的主跌倒事件视频数据集。
- 构建 `fall` / `ADL` 二分类事件评估集。
- 对现有链路做批量处理：

```text
mp4
  -> run_fall_tracking.py
  -> run_fall_pose.py
  -> run_fall_pose_quality.py
  -> run_fall_near_fall.py / 后续 fall_event 检测
  -> 事件级 precision / recall / F1
```

建议标注映射：

| 源目录 | 项目标签 |
|---|---|
| `VideoDataset/Fall/**.mp4` | `fall_event` |
| `VideoDataset/ADL/**.mp4` | `non_fall_adl` |

本地解压检查：

- Fall 事件 MP4：997 个。
- ADL 事件 MP4：1017 个。
- 所有 MP4 可被 `ffprobe` 读取。
- Fall 中有 16 个子目录只有 `metadata.json` 没有 MP4。
- Fall 中有 5 个子目录、ADL 中有 1 个子目录包含多个 MP4。
- 构建 manifest 时应按实际 MP4 文件建样本，并过滤 `timelapse*.mp4`、`all_csvs/`、`all_plots/`。

限制：

- 这是公开实验/模拟型跌倒数据，不等同于真实养老机构老人自然跌倒。
- 不适合训练个体化行为基线。
- 不适合直接证明长期跌倒风险预警效果。

## 3. UR Fall Detection Dataset

来源：

```text
https://fenix.ur.edu.pl/~mkepski/ds/uf.html
```

本地文件：

```text
data/external/ur_fall/raw/
```

当前已完成核心子集：

- 100 个 `*.mp4`。
- 142 个 `*.csv`。
- 官方页面解析出的完整链接清单：`source_links.txt`。

暂缓下载：

- 200 个深度/RGB 图片序列 zip 中，目前只保留 139 个已下载 zip，且未见解压后的图片序列目录。它们不是当前视频姿态链路的必需输入。

本地解压检查：

- `fall-*-cam0.mp4`：30 个。
- `fall-*-cam1.mp4`：30 个。
- `adl-*-cam0.mp4`：40 个。
- `fall-*-acc.csv` / `fall-*-data.csv`：各 30 个。
- `adl-*-acc.csv` / `adl-*-data.csv`：各 40 个。
- 所有 100 个 MP4 可被 `ffprobe` 读取。

推荐用途：

- 快速验证跌倒/ADL 事件判断是否在另一个小数据集上仍能跑通。
- 使用 `fall-*-cam0.mp4` 和 `fall-*-cam1.mp4` 检查多视角差异。
- 使用 `adl-*-cam0.mp4` 做误报分析。
- 使用 `*-acc.csv` 做后续视频 + IMU 融合实验。
- 使用 `urfall-cam0-falls.csv` 和 `urfall-cam0-adls.csv` 做深度特征 baseline 对照。

建议标注映射：

| 文件模式 | 项目标签 |
|---|---|
| `fall-*-cam0.mp4` / `fall-*-cam1.mp4` | `fall_event` |
| `adl-*-cam0.mp4` | `non_fall_adl` |
| `fall-*-data.csv` / `adl-*-data.csv` | 同步帧与传感器摘要 |
| `fall-*-acc.csv` / `adl-*-acc.csv` | 加速度序列 |
| `urfall-cam0-falls.csv` / `urfall-cam0-adls.csv` | 已提取深度图特征和姿态标签 |

限制：

- 数据集规模小，不能单独作为最终指标依据。
- 许可为非商业学术用途，不能按 CC BY 4.0 处理。
- ADL 只有 camera 0，fall 有 camera 0/1，评估时要避免摄像头数量差异造成偏差。

## 推荐实验顺序

1. 先用 LE2I/IMViA `Home_01` 建立固定烟测和跌倒时间窗验证。
2. 用 Figshare 2017 数据集建立主事件评估集。
3. 用 UR Fall 的 MP4/CSV 核心子集做交叉验证。
4. 当前阶段不要强依赖 UR Fall 的深度/RGB 图片序列 zip。
5. 报告中写清楚：这些数据集用于公开模拟/实验跌倒事件验证，真实老人长期照护场景仍需 IPML/Databrary 或自采数据做外部验证。

## 与固定算法路线的关系

这些数据集主要覆盖：

```text
人体检测与跟踪
  -> 人体姿态关键点提取
  -> 关键点质量控制与时序平滑
  -> 近跌倒/跌倒事件检测
  -> 事件级评估
```

它们不能充分覆盖：

```text
个体化行为基线建模
轻量风险融合模型 + 规则校准
真实长期跌倒风险等级评估
```

所以后续仍然需要 TOAGA/GSTRIDE 和真实历史数据分别补足步态链路、风险校准和长期外部验证。

补充说明：

- TOAGA 承担视频步态链路验证。
- GSTRIDE 不含视频，包含 Excel 元数据、足部 IMU CSV、步态分割 CSV 和步态参数 CSV；它承担风险/虚弱/跌倒者标签校准，不能用于验证视频检测、跟踪或 RTMPose 链路。
