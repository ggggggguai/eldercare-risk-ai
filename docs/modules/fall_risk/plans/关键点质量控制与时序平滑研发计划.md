# 跌倒风险模块：关键点质量控制与时序平滑执行计划

## 任务目标

为跌倒风险模块实现“关键点质量控制与时序平滑”这一层。

该层位于固定算法路线中的：

```text
萤石设备或开放平台视频流
  -> 人体检测与跟踪
  -> 人体姿态关键点提取
  -> 关键点质量控制与时序平滑
  -> 步态稳定性分析
  -> 坐站转换能力分析
  -> 近跌倒事件检测
  -> 个体化行为基线建模
  -> 轻量风险融合模型 + 规则校准
  -> 跌倒风险等级 + 置信度 + 可解释风险因子 + 预警动作建议
```

本任务只处理姿态关键点序列质量，不直接输出最终跌倒风险等级、风险分数或预警动作。

## 执行约束

- 测试和脚本必须使用项目 conda 环境。
- 使用 `conda run -n eldercare-ai python -m pytest ...`。
- 不要使用裸 `python`、裸 `pytest`。
- 优先测试驱动开发。
- 不要跳过该层直接从视频或姿态输出最终风险。
- 本阶段使用可解释、可测试的规则/统计 baseline。
- 不引入复杂时序深度模型。
- 不安装新依赖。
- 不运行长耗时视频推理。

## 目标行为定义

### 输入

输入为姿态提取阶段输出的 JSONL。每行表示一人一帧，核心字段包括：

- `frame_id`
- `timestamp_sec`
- `person_id`
- `track_id`
- `bbox`
- `scene_region`
- `pose_confidence`
- `keypoint_quality`
- `keypoints`

### 输出

输出为按 `person_id` + `track_id` 分组处理后的稳定关键点 JSONL。每行仍表示一人一帧，但需要新增：

- 核心关键点质量。
- 每个关键点的有效性标记。
- 插值来源标记。
- 平滑坐标。
- 异常跳变标记。
- 连续低质量帧状态。
- 时间窗质量摘要。

### 边界

本步骤输出的是“可用的、带质量标记的稳定姿态序列”和窗口摘要。

本步骤不做：

- 不计算 `gait_risk_score`。
- 不计算 `sit_stand_risk_score`。
- 不计算 `near_fall_event_score`。
- 不输出 `risk_level`。
- 不输出 `recommended_action`。
- 不输出最终风险事件 JSON。

## 核心算法计划

### 核心关键点集合

第一版核心关键点：

```text
left_shoulder
right_shoulder
left_hip
right_hip
left_knee
right_knee
left_ankle
right_ankle
```

可选增强点：

```text
left_wrist
right_wrist
```

手腕点可用于后续快速扶墙、快速支撑等近跌倒线索，但第一版不要把手腕作为必需核心点。

### 低置信度关键点过滤

默认阈值：

```text
min_keypoint_score = 0.30
```

低于阈值的点：

- `valid = false`
- `source = "low_confidence"`
- 不参与核心质量评分。
- 不参与速度计算。
- 不参与角度计算。
- 不作为平滑强观测输入。

### 单帧质量评分

保留现有 `keypoint_quality`，新增更面向下游动作分析的 `core_keypoint_quality`。

建议公式：

```text
core_keypoint_quality = 0.7 * core_coverage + 0.3 * mean_core_score
```

其中：

- `core_coverage` 为核心关键点中 `score >= min_keypoint_score` 的比例。
- `mean_core_score` 为核心关键点平均置信度。

同时输出：

- `valid_core_count`
- `missing_core_names`
- `quality_state`

建议 `quality_state` 枚举：

```text
usable
low_quality
low_quality_run
missing_core
```

### 连续低质量帧标记

默认阈值：

```text
low_quality_threshold = 0.45
low_quality_run_frames = 3
```

规则：

- 单帧 `core_keypoint_quality < low_quality_threshold` 时，标记为 `low_quality`。
- 连续低质量帧数达到 `low_quality_run_frames` 时，标记为 `low_quality_run`。
- 输出 `low_quality_run_length`。

### 短缺失片段插值

默认参数：

```text
max_interp_gap_frames = 2
```

可根据实际帧率调整为 3，但第一版建议从 2 开始，避免过度补点。

规则：

- 按 `person_id + track_id + keypoint.name` 分组。
- 缺失片段长度小于等于 `max_interp_gap_frames` 时，线性插值。
- 缺失片段过长时，不插值。
- 插值点不伪装为真实模型观测。

插值点字段：

- `source = "interpolated"`
- `valid = true`
- `score` 可沿用较低置信度或新增 `quality_weight`。

长缺失点字段：

- `source = "missing"`
- `valid = false`

### 平滑

第一版建议实现指数平滑。

默认参数：

```text
alpha = 0.4
```

要求：

- 保留原始 `x/y`。
- 输出 `x_smooth/y_smooth`。
- 只对真实有效点和短缺失插值点进行平滑。
- 长缺失后重新观测时重置平滑状态，避免跨遮挡拖影。
- 不用平滑结果覆盖原始坐标。

### 异常跳变检测

默认参数：

```text
jump_threshold_norm = 0.18
```

规则：

- 对归一化坐标计算相邻帧位移。
- 核心关键点单帧位移超过阈值时，标记为异常跳变。
- 若相邻躯干或同侧关节不支持该突变，也应倾向于标记异常。
- 第一版可以先用单点位移阈值做 baseline，再逐步增加骨架一致性判断。

输出字段：

- `is_jump_outlier = true | false`

跳变点处理：

- 不直接删除。
- 降权。
- 不作为平滑强观测。

### 每个时间窗的质量摘要

默认窗口：

```text
window_sec = 1.0
```

或在无可靠 FPS 时使用固定帧数窗口，例如：

```text
window_frames = 10
```

输出：

- `mean_core_keypoint_quality`
- `low_quality_frame_ratio`
- `interpolated_point_ratio`
- `jump_outlier_count`
- `usable_for_gait`
- `usable_for_sit_stand`
- `usable_for_near_fall`

可用性 baseline：

- `usable_for_gait`：左右髋、膝、踝在窗口内覆盖率足够。
- `usable_for_sit_stand`：肩、髋、膝、踝覆盖率足够。
- `usable_for_near_fall`：核心点整体质量可用，且低质量连续段比例不过高。

## 后续模块使用字段

### 步态稳定性模块

使用字段：

- `left_ankle/right_ankle`
- `left_knee/right_knee`
- `left_hip/right_hip`
- `x_smooth/y_smooth`
- `valid`
- `window_quality.usable_for_gait`

用途：

- 步频。
- 步幅。
- 左右步态对称性。
- 重心横向摆动。
- 行走路径偏移。

### 坐站转换模块

使用字段：

- 肩、髋、膝、踝平滑坐标。
- `core_keypoint_quality`
- `window_quality.usable_for_sit_stand`

用途：

- 起身/坐下过程识别。
- 坐站转换时间。
- 躯干前倾角。
- 起身后稳定时间。

### 近跌倒检测模块

使用字段：

- 核心点平滑轨迹。
- `is_jump_outlier`
- `low_quality_run_length`
- `window_quality.usable_for_near_fall`

用途：

- 重心突然横向偏移。
- 身体快速下沉但未倒地。
- 行走中急停并恢复。
- 异常轨迹突变。

```text

```

必须覆盖：

- 小样本验证。
- 可解释输出。
- 单元测试覆盖。
- 与后续步态、坐站、近跌倒模块解耦。
- 作为后续复杂时序模型的对照基线。
