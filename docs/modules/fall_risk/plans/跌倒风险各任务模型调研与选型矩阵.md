# 跌倒风险各任务模型调研与选型矩阵

> 调研更新时间：2026-07-10
>
> 文档性质：模型候选池、选型依据与实验路线，不代表候选模型均已接入或已经在本项目数据上验证。
>
> 上位约束：所有候选都必须服从“检测跟踪 -> 姿态 -> 质量控制 -> 行为分支 -> 个体基线 -> 风险融合”的固定算法路线，不能用端到端视频分类器绕过中间层直接输出最终风险。

## 1. 调研结论

### 1.1 事实

- 当前代码已经有 YOLOv8 + ByteTrack、YOLOv8-pose/RTMPose 适配、规则式姿态质量控制、步态/坐站/近跌倒 baseline 和结构化风险融合接口。
- 研发计划此前给出了 TCN、ST-GCN、Isolation Forest、Transformer AutoEncoder、LightGBM 等方向，但部分位置只写“更高精度检测器”“其他姿态模型”“个体时序模型”；本次已把这些泛称同步替换为可执行短名单，并由本文提供完整候选依据。
- COCO、PoseTrack、NTU RGB+D 等公开基准主要验证通用检测、姿态或动作识别能力；Le2i 等数据主要覆盖已经发生的跌倒。它们都不能单独证明模型具备“老人跌倒前风险预测、近跌倒识别或个体长期风险变化”能力。
- 不同论文的精度通常来自不同数据集、输入分辨率、硬件和测试协议。本文不把论文表格中的单个最高分当成本项目选型结论。

### 1.2 推断

- 当前项目的主要瓶颈不是缺少更大的网络，而是老人域数据、近跌倒正样本、逐人长期基线和跨房间泛化证据不足。
- 小样本阶段，规则/统计特征、轻量 TCN、可解释树模型和正常行为建模比大型 Transformer 更容易形成可信闭环。
- 上游姿态的时间抖动、遮挡缺失和跟踪 ID 切换会向所有下游分支传播。先比较跟踪、姿态和时序平滑，通常比直接替换风险融合模型更有收益。
- 单一模型不可能同时兼顾实时性、遮挡恢复、动作边界、稀有异常和长期个体变化；多样化应体现为不同建模假设的对照，而不是只换模型尺寸。

### 1.3 建议

第一阶段只保留能够形成有效消融的短名单：

1. 跟踪比较 ByteTrack、BoT-SORT、OC-SORT。
2. 姿态比较当前 YOLOv8-pose、RTMPose-S/M、ViTPose-S 或 HRNet-W32 中的一个高精度对照。
3. 平滑比较 EMA、One Euro、Kalman；只有统计滤波仍不能满足时序指标时再训练 SmoothNet。
4. 步态保持“人工特征 baseline + 轻量 TCN 主模型”，新增 ST-GCN++ 或 CTR-GCN 作为骨架图对照。
5. 坐站先做状态机/规则与 TCN 或 MS-TCN++ 对照；有逐帧相位标签后再上 ASFormer。
6. 近跌倒采用“规则候选生成 + 正常行为异常检测 + 有标签分类器”的级联，而不是直接依赖一个大型异常 Transformer。
7. 个体基线先加入稳健 Z 分数、EWMA/CUSUM 和变点检测；融合层比较规则、Logistic、EBM/LightGBM，并强制做概率校准。

## 2. 总体候选图谱

| 任务 | 当前主线 | 应加入的多样化方案 | 当前优先级最高的新增对照 |
|---|---|---|---|
| 人体检测 | YOLOv8 | RTMDet、RT-DETR/D-FINE、开放词汇检测 | RTMDet-tiny 或 RT-DETR 小模型 |
| 多目标跟踪 | ByteTrack | BoT-SORT、OC-SORT、Deep OC-SORT | BoT-SORT、OC-SORT |
| 2D 姿态 | YOLOv8-pose、RTMPose | ViTPose、HRNet/LiteHRNet、RTMO、whole-body | RTMPose-S/M、ViTPose-S |
| 3D 姿态增强 | 暂无 | VideoPose3D、MotionBERT | 仅离线验证，不进入第一版实时主线 |
| 质量控制与平滑 | 置信度过滤、插值、EMA | One Euro、Kalman、Savitzky-Golay、SmoothNet、DeciWatch | One Euro、Kalman |
| 时序表征 | 人工通道、滑窗、TCN 准备 | MS-TCN++、ST-GCN++、CTR-GCN、PoseC3D、Transformer | ST-GCN++ |
| 步态风险 | 规则 baseline、轻量 TCN 计划 | GAM/EBM、LightGBM、ST-GCN++、CTR-GCN、3D 生物力学 | TCN 与 ST-GCN++ 对照 |
| 坐站转换 | 规则/统计 baseline | HMM/状态机、TCN、MS-TCN++、ASFormer、PoseC3D | TCN 或 MS-TCN++ |
| 近跌倒 | 规则突变 | ECOD/Isolation Forest/OC-SVM、Deep SVDD、时序自编码器、ST-GCN++ | ECOD/Isolation Forest/OC-SVM |
| 个体基线 | 滚动均值/标准差/分位数 | EWMA/CUSUM、混合效应模型、PELT/BOCPD、在线异常检测 | EWMA/CUSUM + PELT |
| 风险融合 | 规则、Logistic/LightGBM | EBM/GAM、CatBoost、贝叶斯网络、缺失感知时序融合 | Logistic、EBM、LightGBM |
| 置信度与拒识 | 规则降权 | Platt/Isotonic、温度缩放、保序校准、Conformal prediction | Isotonic + 明确拒识阈值 |

优先级表示“最值得在当前数据和工程条件下比较”，不代表理论上最强。

## 3. 人体检测与跟踪

### 3.1 检测器候选

| 路线 | 代表模型 | 优点 | 主要风险 | 本项目定位 |
|---|---|---|---|---|
| 实时单阶段检测 | Ultralytics YOLO 系列 | 当前代码与权重链路已经打通，部署成本最低 | 更换新代模型会引入环境、导出和权重兼容性变化 | 主 baseline；当前仍固定 YOLOv8，最新代只做独立基准 |
| 实时单阶段检测 | [RTMDet](https://arxiv.org/abs/2212.07784) | 与 MMPose/RTMPose 生态一致，便于形成统一 OpenMMLab 管线 | 需要新增 MMDetection 依赖和输出适配 | 值得比较 tiny/small 规格 |
| 端到端 Transformer 检测 | [RT-DETR](https://arxiv.org/abs/2304.08069) | 不依赖传统 NMS，属于与 YOLO 不同的建模路线 | 低功耗 CPU/边缘设备上未必优于轻量 YOLO | GPU 离线或服务器端精度对照 |
| 改进 DETR 回归 | [D-FINE](https://arxiv.org/abs/2410.13842) | 提供较新的实时 DETR 候选，可验证 DETR 路线是否有收益 | 本项目尚无部署经验，依赖和导出需单独核验 | 后续增强实验，不进入首轮 |
| 开放词汇检测 | [Grounding DINO](https://arxiv.org/abs/2303.05499) | 可用文本提示发现扶手、助行器、家具等新场景对象 | 对固定 `person` 检测过重，开放词汇分数也不能直接当接触证据 | 仅用于场景对象探索，不替代人体检测主线 |

检测器不能只看通用 mAP。项目内必须按老人域标注比较：

- 人体召回率，尤其是坐姿、弯腰、倒地、侧身和下肢遮挡状态。
- 每分钟漏检持续时间、最长连续漏检帧数。
- 小目标和低光场景召回。
- 检测框抖动对姿态裁剪的影响。
- 实际目标设备上的端到端帧率、P95 延迟和显存/内存。

### 3.2 跟踪器候选

[Ultralytics Track 官方文档](https://docs.ultralytics.com/modes/track/)当前列出 ByteTrack、BoT-SORT、OC-SORT、Deep OC-SORT 等多种内置跟踪器，允许在检测器不变时直接比较跟踪策略。

| 跟踪器 | 核心差异 | 适用场景 | 不适用或代价 | 项目建议 |
|---|---|---|---|---|
| [ByteTrack](https://arxiv.org/abs/2110.06864) | 两阶段关联并利用低置信度框 | 固定机位、单老人或少人、实时要求高 | 没有外观 ReID，长遮挡后可能换 ID | 保留为最小开销 baseline |
| [BoT-SORT](https://arxiv.org/abs/2206.14651) | 运动、相机运动补偿和可选 ReID | 遮挡、多人物经过或摄像头轻微移动 | ReID 增加算力；固定单人场景收益可能有限 | 首轮重点对照 |
| [OC-SORT](https://arxiv.org/abs/2203.14360) | 观测中心的运动修正 | 急转、踉跄等非线性运动，且不想承担 ReID 成本 | 长时完全遮挡仍有限 | 首轮重点对照 |
| [Deep OC-SORT](https://arxiv.org/abs/2302.11813) | OC-SORT 加自适应外观信息 | 多人遮挡且 ID switch 是主要错误 | 复杂度和依赖更高 | 仅在 BoT-SORT/OC-SORT 仍频繁换 ID 时启用 |
| FastTracker/TrackTrack | 遮挡回滚或多线索关联，已进入 Ultralytics 新版内置候选 | 新版本探索 | 项目当前环境和长期稳定性需验证 | P2 探索，不应先于前三种 |

跟踪选型指标应是 HOTA/IDF1 的项目内版本，加上 `ID switches/min`、轨迹碎片数、遮挡后恢复时间和下游事件窗口被切断的比例。固定机位单老人场景中，“更复杂”不等于“更好”；如果 ByteTrack 已无 ID 切换，增加 ReID 只会提高延迟。

## 4. 姿态关键点提取

[MMPose Model Zoo](https://mmpose.readthedocs.io/en/latest/model_zoo.html)同时提供 RTMPose、HRNet、LiteHRNet、ViTPose、RTMO、VideoPose3D 和 MotionBERT 等不同路线，适合用统一数据契约做对照。

| 路线 | 代表模型 | 优点 | 局限 | 本项目定位 |
|---|---|---|---|---|
| 轻量 top-down | [RTMPose](https://arxiv.org/abs/2303.07399) | 实时性和精度平衡好，已有 MMPose 适配层 | 依赖人体框；框抖动会传入关键点 | 目标主线，比较 S/M 规格 |
| 单阶段检测+姿态 | YOLO-pose、RTMO | 管线简单，多人时少一次裁剪/调度 | 与 top-down 模型的输入和误差形态不同 | 当前 YOLOv8-pose 保留为工程 baseline；RTMO 作为生态内单阶段对照 |
| 高分辨率热图 | [HRNet](https://arxiv.org/abs/1902.09212)、LiteHRNet | 关键点定位稳定，经典且可解释 | HRNet 较重，低光和遮挡仍需域内验证 | HRNet-W32 精度对照或 LiteHRNet 边缘对照 |
| Vision Transformer | [ViTPose](https://arxiv.org/abs/2204.12484) | 强表征能力，适合作为高精度上限 | 计算量、预训练依赖和部署成本较高 | 离线生成高质量伪标签或精度对照 |
| Whole-body | RTMW、[DWPose](https://arxiv.org/abs/2307.15880) | 更密集的身体、手部关键点，可增强支撑动作线索 | 更多关键点不等于可靠家具接触；计算和 schema 适配更复杂 | 只在“扶墙/扶物”成为核心误差来源后评估 |
| 2D-to-3D lifting | [VideoPose3D](https://arxiv.org/abs/1811.11742)、[MotionBERT](https://arxiv.org/abs/2210.06551) | 可减弱单一视角下角度、深度和尺度混淆 | 室内老人域与训练域差异明显；单目 3D 不是测量真值 | 离线增强与误差分析，不进入第一版告警主线 |

首轮姿态实验建议冻结同一个检测框和跟踪结果，只比较：

```text
YOLOv8-pose baseline
vs RTMPose-S/M
vs ViTPose-S 或 HRNet-W32（二选一作为高精度上限）
```

必须单独报告踝、膝、髋、肩、腕的 PCK/OKS 或人工复核通过率。通用全身平均 AP 会掩盖本项目最重要的下肢错误。还需统计关键点有效覆盖率、连续缺失长度、抖动、左右点互换和跨视角稳定性。

## 5. 关键点质量控制与时序平滑

### 5.1 候选方法

| 类别 | 代表方法 | 优点 | 风险 | 推荐用途 |
|---|---|---|---|---|
| 简单低通 | EMA、移动平均 | 无训练、实现简单、延迟可控 | 强平滑会抹掉踉跄和快速下沉峰值 | 现有 baseline |
| 自适应低通 | [One Euro Filter](https://doi.org/10.1145/2207676.2208639) | 低速时抑制抖动、高速时降低滞后 | 需要按帧率和速度调参 | 首轮实时对照 |
| 状态空间 | Kalman/扩展 Kalman | 可显式建模速度和观测噪声，支持短缺失预测 | 运动模型不匹配时会产生过度外推 | 首轮实时对照 |
| 局部多项式 | Savitzky-Golay | 能保留局部导数，离线特征分析方便 | 非因果窗口有未来信息，边界处理复杂 | 离线特征对照，不直接用于实时告警 |
| 解剖约束优化 | 固定骨长、关节角限制、样条/鲁棒优化 | 可抑制明显不可能姿态 | 错误约束会放大遮挡误差；个体骨长需初始化 | 作为异常修正和质量分，不宜强制覆盖原观测 |
| 学习式后处理 | [SmoothNet](https://arxiv.org/abs/2112.13715) | 专门学习姿态序列去抖，可插在现有估计器之后 | 需要训练域匹配；窗口会引入延迟 | 统计滤波不足后的 P1 对照 |
| 稀疏采样与补全 | [DeciWatch](https://arxiv.org/abs/2203.08713) | 联合降低逐帧姿态推理开销并恢复时序 | 快速近跌倒动作可能落在未采样帧之间 | 仅在姿态推理成为明确性能瓶颈时评估 |

### 5.2 不可只看“更平滑”

平滑层至少同时优化四类指标：

- 抖动：静止片段速度/加速度噪声、关键点加速度误差。
- 保真：人工标注帧上的 PCK/OKS 不应明显下降。
- 事件峰值：快速下沉、横向失衡、腕部快速支撑的峰值保留率。
- 时延：事件峰值时间偏移、在线算法 P95 延迟。

近跌倒检测最怕“看起来更顺，但事件被平掉”。因此 SmoothNet、双向插值和 Savitzky-Golay 只能在严格区分在线/离线协议后比较；使用未来帧的结果不能冒充实时性能。

缺失补全必须带 `mask/source` 进入下游。插值点不能与真实观测同权，连续长缺失不能用模型生成的平滑骨架假装成可靠证据。

## 6. 关键点时序表征与通用骨干

该层只生成可供步态、坐站和近跌倒分支使用的表征，不直接输出最终风险。

| 家族 | 代表方法 | 建模能力 | 数据与工程要求 | 建议 |
|---|---|---|---|---|
| 人工时序特征 | 坐标、骨长归一化、速度、加速度、角度、对称性、质量 mask | 可解释、适合小样本 | 依赖特征设计和视角归一化 | 所有深度模型都必须保留的 baseline |
| 时间卷积 | TCN、[MS-TCN++](https://arxiv.org/abs/2006.09220) | 局部到长时序模式、训练稳定、可做相位分割 | 需要窗口或逐帧标签 | 当前最合适的通用主干 |
| 骨架图卷积 | [ST-GCN](https://arxiv.org/abs/1801.07455)、2s-AGCN、[CTR-GCN](https://arxiv.org/abs/2107.12213)、ST-GCN++ | 显式利用关节拓扑，可输入 joint/bone/motion 多流 | 需要统一骨架拓扑；预训练动作域与老人风险域不同 | ST-GCN++/CTR-GCN 作为强对照 |
| 姿态热图视频 | [PoseC3D](https://arxiv.org/abs/2104.13586) | 对关键点坐标噪声有一定容忍，保留空间不确定性 | 生成热图占存储和算力 | 适合坐站/近跌倒离线增强 |
| RGB 视频 | SlowFast、[VideoMAE](https://arxiv.org/abs/2203.12602) | 能利用家具、支撑接触、外观和场景信息 | 数据量、隐私和算力要求高，可解释性较弱 | 只作为辅助证据分支，不替代姿态主线 |
| 时序 Transformer | MotionBERT 表征、轻量 skeleton Transformer | 长时依赖和预训练迁移 | 小样本易过拟合，延迟和校准成本高 | 数据充足后的 P2 对照 |

[MMAction2 骨架模型库](https://mmaction2.readthedocs.io/en/latest/model_zoo/skeleton.html)提供 AGCN、PoseC3D、ST-GCN 和 ST-GCN++ 的统一配置和预训练权重，可减少重复实现成本。预训练权重只提供初始化，不是本项目有效性的证据。

建议统一缓存以下四流输入，避免每个任务重复预处理：

```text
joint:        归一化关节坐标 + confidence + valid mask
joint-motion: 一阶/二阶时间差分
bone:         相邻关节向量、骨长和角度
bone-motion:  骨向量时间差分
```

质量 mask、插值来源和时间戳必须保留为独立通道。不同帧率的视频应按真实时间重采样，不能把“每帧位移”直接当“速度”。

## 7. 步态稳定性分析

### 7.1 多路线候选

| 路线 | 输入 | 可输出内容 | 优点 | 局限与定位 |
|---|---|---|---|---|
| 生物力学特征 + 规则 | 2D/3D 关键点、轨迹 | 步速 proxy、步频、对称性、髋摆、停顿、转身 | 最可解释，适合建立数据质量和标注基线 | 2D 单目不能声称得到真实米制步幅；必须保留 |
| 特征 + GAM/EBM/LightGBM | 聚合步态特征 | 风险概率和特征贡献 | 小样本友好，可做非线性关系 | 需要逐人划分和校准；适合作为 TCN 之外的模型对照 |
| 轻量 TCN | `[T,V,C]` 或特征序列 | 窗口级 gait risk | 训练稳定、实时性好、与当前计划一致 | 对骨架拓扑利用不如 GCN；主预测模型 |
| ST-GCN++/CTR-GCN | joint/bone/motion 多流 | 窗口级分类或回归 | 显式关节关系，能学左右不对称和协调性 | 需要更多域内样本；增强实验 |
| PoseC3D | 关键点热图序列 | 步态/转身动作表征 | 对坐标量化误差更稳健 | 计算和存储更大；离线对照 |
| 3D 姿态 + 动力学 | 3D skeleton 或多视角视频 | 更接近真实角度、步长和关节运动 | 可用于验证 2D proxy 是否可信 | 单目 lifting 偏差不能当临床真值；研究验证路线 |

[OpenCap](https://doi.org/10.1371/journal.pcbi.1011462)展示了多视角智能手机视频恢复三维运动学/动力学的研究路线，但它依赖特定采集和标定条件。它更适合小规模实验室验证，不适合直接替换居家单目实时主线。

### 7.2 明确排除的误用

GaitSet、GaitPart、GaitGL 等轮廓步态模型主要服务“根据走路外观识别人”的 gait recognition。它们可以提供表征，但其原始目标不是步态稳定性或跌倒风险。没有风险标签微调和域内验证时，不应把身份识别准确率写成风险预测能力。

### 7.3 推荐实验

```text
A. 人工特征规则 baseline
B. 人工特征 + EBM/LightGBM
C. `[T,V,C]` + 轻量 TCN（主线）
D. joint/bone/motion + ST-GCN++（增强）
```

除 AUROC/F1 外，必须按人报告灵敏度、特异度、Brier score，并对慢走、转身、遮挡、助行器和非正面视角单独分层。窗口随机切分会把同一人的相邻片段泄漏到训练和测试，必须按人或按家庭划分。

## 8. 坐站转换能力分析

坐站任务同时包含“事件定位、相位分割、局部能力评分”三个子问题，单一窗口分类标签不足以覆盖全部需求。

| 路线 | 适合标签 | 能解决的问题 | 局限 | 建议 |
|---|---|---|---|---|
| 状态机/HMM | 少量事件边界、姿态阈值 | 站/过渡/坐相位、持续时间、失败尝试 | 阈值受视角和家具高度影响 | 保留为强解释 baseline |
| TCN | 事件级或逐帧标签 | 起坐分类、局部风险回归 | 边界精度依赖窗口 | 当前首选学习模型 |
| MS-TCN++ | 逐帧相位标签 | 多阶段时序分割、抑制过分割 | 标注成本高于事件标签 | 有相位标签后优先比较 |
| [ASFormer](https://arxiv.org/abs/2110.08568) | 较充足逐帧标签 | 长时动作分割和阶段关系 | Transformer 对小样本敏感 | P2 相位分割对照 |
| ST-GCN++/CTR-GCN | 骨架序列 + 事件标签 | 学习关节协调和躯干-下肢关系 | 对家具接触不可见 | 骨架增强实验 |
| PoseC3D/SlowFast/VideoMAE | RGB 或姿态热图 | 扶手、椅子和身体接触上下文 | 隐私、算力、误用场景纹理的风险 | 仅作为支撑使用辅助分支 |
| 3D lifting/多视角 | 2D pose + 标定 | 更可靠的髋高度、躯干角和关节伸展 | 部署复杂，单目深度误差大 | 小规模验证，不作为主线门槛 |

推荐先把输出拆开：

```text
phase probabilities -> 事件开始/结束与起坐类型
event features       -> duration/attempts/sway/support proxy
local risk model     -> sit_stand_risk_score + confidence
```

“疑似使用支撑”需要腕部轨迹与墙面/家具检测或分割共同提供证据。只根据手腕靠近画面边缘不能可靠判断扶墙，必须继续使用 `support_contact_proxy` 语义。

## 9. 近跌倒事件检测

近跌倒不是普通动作分类：正样本稀少、边界短、动作类型多、误报代价高，并且必须证明动作随后恢复、没有进入稳定倒地状态。

### 9.1 候选层次

| 层次 | 代表方法 | 适用数据 | 优点 | 主要风险 |
|---|---|---|---|---|
| 规则候选生成 | 速度/加速度峰值、下沉-恢复、横移-恢复、急停、腕部支撑 proxy | 无训练数据 | 高可解释，可保证证据窗口 | 阈值跨视角泛化有限 |
| 统计变化检测 | CUSUM、EWMA、Matrix Profile、变点检测 | 大量正常序列 | 能发现局部突变，不依赖近跌倒正样本 | 只能说明“异常”，不能自动命名事件 |
| 经典单类模型 | [Isolation Forest/LOF/One-Class SVM](https://scikit-learn.org/stable/modules/outlier_detection.html)、[ECOD 等 PyOD 模型](https://pyod.readthedocs.io/en/latest/) | 以正常窗口为主 | 训练快，适合特征级消融 | 高维时序需先聚合或编码；污染率敏感 |
| 深度单类 | [Deep SVDD](https://arxiv.org/abs/1802.06360) | 较多正常窗口 | 学习正常表征边界 | 可能把罕见但正常的坐下/弯腰当异常 |
| 重构/预测 | TCN-AE、LSTM-AE、[TranAD](https://arxiv.org/abs/2201.07284)、[Anomaly Transformer](https://arxiv.org/abs/2110.02642) | 长期正常序列 | 可建模多变量时间关系 | 大模型可能重构异常；分数难校准，实时窗口有延迟 |
| 有监督骨架分类 | TCN、ST-GCN++、CTR-GCN | 足量近跌倒事件标签 | 能学习事件类型，便于端到端评估 | 真实正样本不足时极易过拟合表演数据 |
| 有监督视频分类 | PoseC3D、SlowFast、VideoMAE | 足量 RGB 事件片段 | 能看见墙面/家具接触和场景 | 隐私、算力和背景捷径问题最明显 |

### 9.2 推荐级联

```text
质量门控
  -> 高召回规则/变点候选窗口
  -> 正常行为异常分数（ECOD/IF/OC-SVM 三选一或小型集成）
  -> 有标签 TCN/ST-GCN++ 事件分类器（数据足够时）
  -> “恢复且未倒地”约束
  -> near_fall_event_score + event_type + evidence
```

这个结构允许在没有足够正样本时先落地，也避免异常检测分数直接被解释为近跌倒概率。异常模型的训练集要做“正常污染”审计；弯腰、快速坐下、拾物、锻炼和宠物/他人遮挡必须作为 hard negative。

评价指标应包括事件级 recall、每人每天误报数、平均提前/检测延迟、事件边界 IoU、恢复约束正确率和人工复核工作量。帧级 accuracy 在极度类别不平衡下没有决策价值。

## 10. 个体化行为基线

| 路线 | 代表方法 | 优点 | 风险 | 推荐阶段 |
|---|---|---|---|---|
| 稳健统计 | 中位数/MAD、分位数带、滚动 Z 分数 | 可解释、冷启动容易 | 对趋势和季节性建模有限 | P0，替代单纯均值/标准差的脆弱部分 |
| 在线控制图 | EWMA、CUSUM | 对持续小幅恶化敏感，计算轻 | 阈值需控制误报；事件污染会拖动基线 | P0 |
| 分层/混合效应模型 | 人群固定效应 + 个体随机效应 | 冷启动时可借用人群先验，随后个体化 | 假设和特征分布需检查 | P1，适合样本人数增加后 |
| 离线变点检测 | [ruptures: PELT/Binseg/Window](https://centre-borelli.github.io/ruptures-docs/) | 能定位状态转折，而不只给异常分 | 通常需要批量历史窗口 | P0/P1，用于周级趋势复盘 |
| 在线变点检测 | BOCPD、ADWIN 等 | 可随新数据更新 | 超参数和稳定性验证更复杂 | P1 |
| 在线异常模型 | Half-Space Trees、增量统计模型 | 长期流式更新，内存可控 | 基线漂移时可能把恶化吸收为“新正常” | P1，必须有冻结和重置策略 |
| 状态空间/高斯过程 | 动态线性模型、GP 趋势 | 能表达噪声、趋势和不确定性 | 计算与实现复杂，解释门槛更高 | P2 |
| 个体微调/元学习 | personalized encoder、few-shot adaptation | 理论上能学习个体动作模式 | 当前人数和长期标签不足 | 暂不优先 |

个体基线必须定义以下策略，否则换模型没有意义：

- 冷启动：最初 7-14 天只建立分布，不输出高置信长期恶化结论；具体天数需由采集频率验证。
- 污染保护：近跌倒、高风险坐站、明显低质量窗口不能直接写回“正常基线”。
- 时间结构：工作日/周末、昼夜、房间和辅助器具状态分层，避免把作息差异当恶化。
- 漂移控制：区分健康状态的真实缓慢变化、摄像头移动和模型版本变更。
- 版本隔离：上游姿态或特征版本变化后不能无条件沿用旧基线分布。

推荐输出不只是一个偏离分：

```text
current_value
expected_range
deviation_direction
deviation_magnitude
change_point_time
baseline_support_days
data_quality
```

## 11. 多模态风险融合、校准与拒识

### 11.1 融合模型候选

| 家族 | 代表模型 | 优点 | 局限 | 项目定位 |
|---|---|---|---|---|
| 规则评分卡 | 加权规则、强事件覆盖 | 最透明，可编码紧急事件和最低质量门槛 | 权重人工、交互有限 | 必须保留的安全 baseline |
| 线性概率模型 | Logistic Regression | 小样本、可校准、系数可审计 | 非线性和高阶交互有限 | 首轮主对照 |
| 可解释非线性 | GAM、[Explainable Boosting Machine](https://interpret.ml/docs/ebm.html) | 能学习单调/非线性特征效应并展示贡献 | 仍需控制交互和过拟合 | 优先于黑盒深网络的增强路线 |
| 梯度提升树 | LightGBM、CatBoost、XGBoost | 处理非线性、缺失值和特征交互能力强 | 概率常需再校准，解释不是因果 | 当前结构化特征强对照 |
| 概率图模型 | 贝叶斯网络、动态贝叶斯网络 | 能表达条件依赖和证据缺失 | 结构学习和先验设计成本高 | 适合专家规则较稳定后的 P1/P2 实验 |
| 时序多模态网络 | TCN/GRU/Transformer fusion | 能利用分支随时间的变化和跨模态交互 | 数据需求大、校准和缺失模态处理困难 | 数据充分后的增强，不作为比赛单点依赖 |

融合输入应使用每个分支的原始特征、局部分数、质量、缺失 mask 和版本，而不是只拼四个风险分数。模型不能把“缺失”默认解释为“正常”。

### 11.2 概率校准和拒识是必选项

[scikit-learn 概率校准指南](https://scikit-learn.org/stable/modules/calibration.html)提供 Platt/sigmoid 和 isotonic 等校准方法。候选模型的 AUROC 较高不代表 `0.8` 真的是 80% 风险；最终风险等级和动作建议依赖阈值，因此必须比较：

- Brier score、log loss 和 reliability diagram/ECE。
- Sigmoid、isotonic；深度模型再比较 temperature scaling。
- 按人、按场景和按模型版本的校准稳定性。
- 低质量、模态冲突、输入超出训练分布时的 `manual_review`/`observe` 拒识率。

Conformal prediction 可作为覆盖率控制实验，但它依赖交换性等假设；时间漂移和个体差异下需要分组或滚动校准，不能直接承诺固定临床覆盖率。

### 11.3 防止标签泄漏

- 按窗口随机划分会泄漏同一人、同一事件和相邻帧，禁止用于最终结论。
- 同一原视频切出的片段必须在同一数据分区。
- 个体历史聚合特征只能使用预测时点之前的数据。
- 阈值、校准器和个体基线参数只能在训练/验证数据确定，测试集不能反复调参。

## 12. 数据条件与模型启用门槛

| 任务 | 最低可用标签 | 数据较少时 | 数据增加后 |
|---|---|---|---|
| 检测/姿态 | 人框、关键点、遮挡/可见性 | 预训练模型 + 域内评测/少量微调 | 分场景微调、蒸馏和伪标签 |
| 平滑 | 稀疏人工关键点 + 事件峰值时间 | One Euro/Kalman | SmoothNet 或学习式修复 |
| 步态 | 人级风险或功能评估 + 行走窗口 | 特征规则、EBM/LightGBM、小 TCN | ST-GCN++/CTR-GCN、多任务学习 |
| 坐站 | 事件边界、类型、耗时/失败/支撑标签 | 状态机、事件 TCN | MS-TCN++/ASFormer 相位分割 |
| 近跌倒 | 大量正常窗口 + 少量事件及 hard negative | 规则 + 单类模型 | TCN/ST-GCN++、PoseC3D、半监督学习 |
| 个体基线 | 每人连续多日特征和质量元数据 | 稳健统计、EWMA/CUSUM | 混合效应、变点/状态空间模型 |
| 融合 | 人级、日级或事件级风险结局 | 规则、Logistic、EBM | GBDT、时序多模态模型 |

这里不设置伪精确的“样本数达到 N 就可上 Transformer”。有效样本量取决于独立老人数量、事件多样性、家庭/机位覆盖和标签噪声。1000 个来自同一人的相邻窗口，远不如 100 个来自不同老人和不同机位的独立事件。

## 13. 分阶段实验矩阵

### 13.1 P0：不新增大量标签即可做

1. 固定 YOLOv8 检测，比较 ByteTrack/BoT-SORT/OC-SORT。
2. 固定同一人体框，比较 YOLOv8-pose/RTMPose-S/M/高精度姿态对照。
3. 在同一原始关键点上比较 EMA/One Euro/Kalman，并测事件峰值保留和延迟。
4. 对当前步态、坐站、近跌倒特征补充 EBM/LightGBM、ECOD/IF/OC-SVM 的离线交叉验证。
5. 个体基线加入 MAD、EWMA/CUSUM 和 PELT 离线变点分析。
6. 融合层比较规则、Logistic、EBM、LightGBM，并统一做校准。

### 13.2 P1：补齐动作/事件标签后

1. 轻量 TCN 步态主模型与 ST-GCN++ 对照。
2. TCN/MS-TCN++ 坐站事件/相位模型。
3. 近跌倒 TCN/ST-GCN++ 有监督分支，与规则+单类模型级联。
4. SmoothNet 时序修复，并验证不同姿态模型之间的迁移。
5. 人群先验 + 个体随机效应的分层基线模型。

### 13.3 P2：数据和硬件满足后

1. ViTPose/RTMW/DWPose 离线伪标签与 whole-body 辅助分支。
2. MotionBERT/VideoPose3D 的 3D 姿态增强。
3. PoseC3D/VideoMAE 的场景接触辅助证据。
4. ASFormer 坐站相位分割、Transformer 异常检测和时序多模态融合。

每轮只替换一个模块并缓存上游结果。禁止一次同时换检测器、姿态、平滑和风险模型后把差异归因给某个模型。

## 14. 统一选型评分表

每个候选在进入代码前都要填写下表，不能只写“论文精度更高”。

| 维度 | 必答问题 |
|---|---|
| 任务匹配 | 原任务是检测、身份识别、动作分类、异常检测还是风险预测？与本任务差多少层？ |
| 输入契约 | 接受 RGB、bbox、2D/3D skeleton、热图还是聚合特征？如何映射当前 JSONL？ |
| 数据需求 | 是否需要逐帧、事件级、个体长期或临床结局标签？ |
| 域适配 | 预训练数据是否包含老人、低光、遮挡、坐姿、倒地和居家机位？ |
| 实时性 | 目标硬件上的端到端 FPS、P50/P95 延迟、内存和启动时间是多少？ |
| 质量与缺失 | 是否接收 confidence/mask？长缺失、ID switch 和低质量窗口如何处理？ |
| 可解释性 | 能否输出风险因子、证据时间窗和拒识原因？ |
| 校准 | 分数是否可校准为决策概率？跨人/跨场景稳定吗？ |
| 工程成熟度 | 官方实现、权重、导出路径、依赖冲突和维护状态如何？ |
| 许可与隐私 | 代码、权重和数据许可是否允许目标用途？是否必须保留 RGB？ |

## 16. 当前推荐组合

在现有项目阶段，综合可落地性、可解释性、小样本和演示稳定性，推荐组合仍然是：

```text
YOLOv8/RTMDet 对照 + ByteTrack/BoT-SORT/OC-SORT 对照
  -> RTMPose 主线 + YOLOv8-pose 工程 baseline + ViTPose/HRNet 精度上限
  -> confidence/mask + One Euro/Kalman + 事件峰值保护
  -> 人工四流特征 + 轻量 TCN
  -> 步态：规则/EBM + TCN，ST-GCN++ 增强
  -> 坐站：状态机 + TCN/MS-TCN++
  -> 近跌倒：规则候选 + 单类异常 + 可选 ST-GCN++
  -> 个体基线：MAD + EWMA/CUSUM + PELT
  -> 规则安全覆盖 + Logistic/EBM/LightGBM + 概率校准 + 低质量拒识
```

这套组合比直接增加一个 Risk Fusion Transformer 更适合当前阶段：它能逐层验证误差、保留解释因子，也能在某个学习模型尚未具备数据条件时继续运行完整链路。Transformer、RGB 视频分支和单目 3D 应作为有明确假设和进入门槛的增强实验，而不是为了“模型看起来多”而并列堆叠。
