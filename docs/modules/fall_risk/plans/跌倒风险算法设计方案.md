# 跌倒风险算法设计方案

## 一、方案定位

本方案面向居家养老、社区养老和独居老人安全监护场景，构建一套基于萤石视频设备和多模态 AI 的老人跌倒风险前置预警算法。系统并非只在老人已经跌倒后进行识别，而是通过持续分析老人的步态、姿态、坐站能力、近跌倒事件和个人行为变化，提前发现跌倒风险升高的信号，实现“跌倒前预测、失衡中预警、跌倒后响应”的完整闭环。

算法目标包括：

- 识别老人日常行走中的步态不稳、拖步、小碎步、转身不稳等风险信号。
- 识别起身困难、坐下不稳、站立后摇晃等坐站转换能力下降信号。
- 识别踉跄、快速扶墙、身体快速下沉、重心异常偏移等近跌倒事件。
- 建立个体化行为基线，判断老人当前状态是否相对自身历史状态异常偏离。
- 输出跌倒风险等级、综合风险分数、模型置信度、可解释风险因子和预警动作建议。

## 二、算法整体流程

整体算法流程如下：

```text
萤石视频流
  ↓
人体检测与跟踪
  ↓
姿态关键点提取
  ↓
关键点质量控制与时序平滑
  ↓
关键点时序特征构建
  ↓
步态稳定性分析
  ↓
坐站转换能力分析
  ↓
近跌倒事件检测
  ↓
个体化行为基线建模
  ↓
多模态风险融合
  ↓
跌倒风险等级 + 置信度 + 风险因子 + 预警建议
```

该流程将原始视频逐步转化为结构化人体轨迹、姿态骨架序列、行为时序特征和风险评分结果。算法设计重点不是单帧图像分类，而是基于连续时间窗口分析老人身体状态和行为趋势。

## 三、人体检测与跟踪

系统首先从萤石摄像头视频流中进行人体检测，识别画面中的老人位置，输出人体框、中心点、检测置信度和所在场景区域。随后通过目标跟踪算法保持同一老人在连续视频帧中的身份一致，形成稳定的人体运动轨迹。

该模块主要提取以下信息：

- 老人当前所在位置。
- 老人运动方向和移动速度。
- 老人在不同区域的停留时间。
- 是否进入床边、走廊、卫生间门口等高风险区域。
- 是否出现长时间静止、突然倒地或离开画面的情况。

人体检测与跟踪模块的作用是将原始视频转化为可计算的运动轨迹数据，为后续姿态估计、行为识别和风险融合提供基础输入。

## 四、姿态关键点提取

在检测到老人所在人体区域后，算法进一步提取人体姿态关键点，包括头部、肩部、肘部、手腕、髋部、膝盖和脚踝等人体骨架点。每一帧姿态数据可以表示为：

```text
K_t = {p_1, p_2, ..., p_n}
```

其中，`p_i` 表示第 `i` 个关键点的位置坐标和置信度。

姿态关键点能够描述老人身体结构和动作变化，例如：

- 躯干是否明显前倾。
- 髋部中心是否快速下降。
- 身体重心是否突然横向偏移。
- 左右下肢运动是否对称。
- 膝盖和脚踝运动是否存在拖步、小碎步特征。
- 手部是否快速靠近墙面、桌面或家具，形成疑似扶物动作。

相比直接使用整张图像进行分类，姿态关键点具有更强的可解释性，也更适合分析老人长期身体能力变化。

## 五、关键点质量控制与时序平滑

居家场景中常见低光、遮挡、侧身、家具遮挡、下肢出画和关键点抖动等问题，因此算法需要对原始姿态关键点进行质量控制和时序平滑，避免单帧误检直接触发误报。

主要处理包括：

- 低置信度关键点过滤。
- 短时缺失关键点插值。
- 关键点异常跳变检测。
- 人体尺度归一化。
- 关键点坐标时序平滑。
- 质量 mask 和窗口质量摘要生成。

质量控制后的姿态序列用于后续步态、坐站和近跌倒分析。对于关键点质量不足的窗口，系统应降低模型置信度或进入人工复核/观察逻辑，而不是简单判断为高风险。

## 六、关键点时序特征构建

算法将连续多帧姿态关键点构造成时序窗口样本：

```text
[T, V, C]
```

其中：

- `T` 表示时间窗口长度。
- `V` 表示人体关键点数量。
- `C` 表示特征通道，包括坐标、置信度、速度、加速度、有效性标记等。

在该时序窗口上，算法提取多类行为特征：

- 步速、步频和步幅。
- 左右下肢运动对称性。
- 髋部中心位移和重心横向摆动。
- 躯干前倾角和身体倾斜程度。
- 起身耗时、坐下耗时和站稳时间。
- 身体快速下沉、异常急停和踉跄恢复。
- 手部快速扶墙、扶桌或扶家具的疑似动作。

该模块相当于将原始视频转化为可供模型理解的行为时序特征，为后续各风险分支提供统一输入。

## 七、步态稳定性分析

步态稳定性是跌倒风险判断的核心指标。算法通过分析老人行走过程中髋、膝、踝等关键点的周期性变化，判断其行走能力是否下降。

重点分析指标包括：

- 步速是否明显下降。
- 步幅是否变小。
- 步频是否异常。
- 左右下肢运动是否不对称。
- 身体横向摆动是否增大。
- 是否存在拖步、小碎步或频繁急停。
- 转身是否缓慢、不稳或摇晃明显。

算法可以采用“人工特征 + 时序模型”的组合方式。人工特征用于解释风险原因，时序模型如 TCN、ST-GCN 或轻量 Transformer 用于学习连续动作中的异常模式。

判断示例：

```text
步速下降
+ 左右脚运动不对称
+ 重心横向摆动增大
= 步态稳定性风险升高
```

输出结果包括：

- `GaitRiskScore`：步态风险分数。
- `GaitRiskFactors`：步态风险原因，例如步速下降、重心摆动增大、转身不稳等。

## 八、坐站转换能力分析

老人从坐姿到站姿、从站姿到坐姿的过程能够反映下肢力量、平衡能力和身体控制能力，是跌倒风险的重要先兆。

算法重点分析以下指标：

- 起身耗时。
- 坐下耗时。
- 起身失败次数。
- 起身过程中的躯干前倾角。
- 髋部高度变化速度。
- 起身后身体摇晃程度。
- 站稳所需时间。
- 是否借助手部支撑。

判断示例：

```text
起身时间明显变长
+ 起身后身体摇晃
+ 多次尝试才站起
= 坐站转换能力下降
```

输出结果包括：

- `SitStandRiskScore`：坐站转换风险分数。
- `SitStandRiskFactors`：坐站风险原因，例如起身耗时较长、起身后摇晃、疑似借助支撑等。

## 九、近跌倒事件检测

近跌倒事件是本方案区别于传统跌倒检测的重要部分。近跌倒指老人尚未真正摔倒，但已经出现明显失衡，例如踉跄、快速扶墙、身体突然下沉、重心大幅偏移、急停后恢复等。

算法通过时序异常检测识别这些事件，重点关注：

- 姿态速度突变。
- 身体高度快速下降。
- 重心横向快速偏移。
- 行走节律突然中断。
- 手部快速靠近墙面或家具。
- 异常动作后短时间恢复。

近跌倒检测可以采用规则突变检测、Isolation Forest、One-Class SVM 或 Transformer AutoEncoder 等方法。系统先学习正常行为模式，再识别偏离正常模式的异常片段。

输出结果包括：

- `NearFallEventScore`：近跌倒事件分数。
- `EventType`：事件类型，例如踉跄、快速扶墙、身体快速下沉、急停恢复等。
- `EvidenceWindow`：证据时间窗，便于回放和复核。

## 十、个体化行为基线建模

不同老人的正常活动能力差异较大，因此系统不能只使用统一阈值判断所有老人。方案为每位老人建立个人行为基线，将当前状态与其自身历史状态比较。

基线特征包括：

- 平均步速。
- 日常步幅。
- 坐站平均耗时。
- 夜间起床次数。
- 日均活动量。
- 常见活动区域。
- 转身稳定性。
- 近跌倒事件频率。

基线偏离判断示例：

```text
当前步速较个人历史下降 25%
+ 当前坐站耗时较历史增加 35%
+ 近 7 天夜间活动明显增加
= 个体化跌倒风险升高
```

输出结果包括：

- `BaselineDeviationScore`：个体基线偏离分数。
- `DeviationFactors`：偏离原因，例如步速下降、夜间活动增加、坐站耗时变长等。

个体化基线建模能够降低统一阈值带来的误判，使系统更关注老人相对自身状态的风险变化。

## 十一、多模态风险融合

最后，算法将多个子模块的输出进行融合，形成最终跌倒风险判断。

融合输入包括：

- 步态风险分数。
- 坐站转换风险分数。
- 近跌倒事件分数。
- 个体基线偏离分数。
- 场景风险分数。
- 活动节律风险分数。
- 疑似跌倒或长时间静止信号。
- 关键点质量与模型置信度。

融合模型可以采用规则评分卡、Logistic Regression、LightGBM 或轻量神经网络。规则评分卡用于保证系统可解释性，机器学习模型用于提升复杂场景下的综合判断能力。

最终输出包括：

```text
risk_level：0-4 级跌倒风险等级
risk_score：综合风险分数
confidence：模型置信度
risk_factors：可解释风险原因
evidence_window：证据时间窗
recommended_action：建议动作
```

风险等级定义如下：

| 风险等级 | 含义 | 建议动作 |
|---:|---|---|
| 0 级 | 正常 | 仅记录 |
| 1 级 | 低风险 | 持续观察 |
| 2 级 | 中风险 | 建议提醒老人或家属关注 |
| 3 级 | 高风险 | 建议通知家属或社区人员 |
| 4 级 | 紧急风险 | 疑似跌倒或长时间倒地，建议立即告警 |

## 十二、硬件设备需求说明

本方案硬件设备主要用于视频采集、边缘计算、数据传输、算法推理和结果展示。硬件配置以“居家可部署、低侵入、可扩展、便于接入萤石开放平台”为原则。

本方案建议优先选用 **萤石 H6c Pro 2K+ 室内云台摄像机，型号 CS-H6c-R105-1J4WF** 作为主要视频采集设备。该型号具备 4MP / 2K+ 分辨率、最高 2560 × 1440 视频输出、最高 30fps 帧率、H.264/H.265 编码、RJ45 有线网络接口、2.4G/5G 双频 Wi-Fi、红外夜视和智能彩色夜视能力，适合室内老人活动监测、姿态关键点提取和夜间风险识别。

像素与录像存储选型建议：

- **像素选择：优先选择 4MP。** 3MP 对普通监控足够，但在较宽房间、下肢动作较小或老人距离摄像头较远时，脚踝、膝盖等关键点细节容易不足；5MP 能提供更多细节，但会增加带宽、存储和推理压力，比赛原型与居家部署的性价比不如 4MP。4MP / 2K+ 在姿态关键点清晰度、帧率、码流压力和成本之间更均衡。
- **录像存储：以电脑或边缘服务器本地保存为主。** 摄像头只负责采集视频流，录像文件、风险事件片段、结构化特征和标注素材统一保存到项目电脑或服务器，便于后续算法训练、复核、标注和报告整理。这样不依赖摄像头存储卡容量，也更方便统一管理数据。
- **摄像头 microSD 卡：可选，不作为必需项。** 如果视频流已经稳定保存到电脑或服务器，摄像头本地卡可以不配；若希望断网、电脑未开机或采集程序异常时仍有兜底录像，可配置 32GB 或 64GB 高耐久 microSD 卡作为备份。只有在需要摄像头独立长时间连续录像时，才考虑 128GB、256GB 或 512GB。

本地电脑或服务器存储建议如下：

| 使用场景 | 建议存储空间 | 说明 |
|---|---:|---|
| 算法演示、只保存风险事件片段 | 100GB 以内 | 主要保存少量测试视频、事件片段和输出结果 |
| 日常测试、短期视频回放和误报复核 | 500GB 左右 | 适合保存多场景、多轮测试数据 |
| 多天连续采集、数据标注和模型验证 | 1TB 或以上 | 推荐配置，便于积累训练、验证和标注素材 |

建议配置如下：

| 边缘计算设备或本地服务器 | 1 台 | 家庭网关、社区养老站或实验室服务器 | 运行人体检测、姿态估计和风险融合算法 |

摄像头部署建议如下：

| 部署区域 | 建议设备 | 监测重点 |
|---|---|---|
| 客厅 | 1 台 H6c Pro 2K+ | 日常行走、转身、久坐、活动量变化 |
| 卧室床边 | 1 台 H6c Pro 2K+ | 起夜、下床、床边停留、夜间跌倒风险 |
| 走廊 | 1 台 H6c Pro 2K+ | 连续行走、拖步、小碎步、步态稳定性 |
| 卫生间门口 | 1 台 H6c Pro 2K+ | 高风险区域停留、湿滑环境下异常静止和跌倒风险 |

H6c Pro 2K+ 是云台摄像机，具备较大的水平和垂直旋转范围，便于安装调试和覆盖房间主要活动区域。但在算法采集和风险识别过程中，应尽量固定预置位，关闭自动巡航、自动追踪或自动变焦追踪，避免相机视角频繁变化影响人体轨迹、场景区域和步态稳定性特征计算。云台能力主要用于安装调试、人工查看和非算法巡检，不应作为步态分析阶段的常态拍摄方式。

部署时应注意：

- 避免正对隐私敏感区域。
- 优先覆盖人体轮廓、下肢动作和主要通行区域。
- 保证画面中老人身体关键部位尽量完整可见。
- 尽量减少强逆光、遮挡和过低机位造成的关键点识别困难。
- 在卧室、卫生间门口等高风险区域，应重点保证夜间成像能力。

边缘计算设备主要承担视频解码、人体检测、姿态估计、时序特征计算和风险融合推理任务。若部署在社区或机构场景，可由一台服务器同时接入多路摄像头；若部署在家庭场景，可采用轻量边缘计算盒或本地小型主机完成实时推理。

## 十三、方案特点

本方案的核心优势包括：

- 不依赖老人主动佩戴设备，降低使用门槛。
- 不只识别已经发生的跌倒，而是识别跌倒前的风险信号。
- 融合步态、坐站、近跌倒、个体基线和场景风险，实现多维判断。
- 输出可解释风险因子和证据时间窗，便于家属、社区人员或平台系统复核。
- 可与萤石开放平台视频流和设备数据适配，具备后续工程落地可行性。

综上，本方案通过“人体检测跟踪 + 姿态关键点分析 + 行为时序建模 + 个体基线比较 + 多模态风险融合”的算法路线，实现对老人跌倒风险的前置识别和分级预警。系统能够从日常行为中发现步态退化、起身困难、近跌倒和活动节律异常等早期风险信号，为居家养老场景下的安全监护提供智能化算法支撑。
