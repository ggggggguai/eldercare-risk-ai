# 算法事件输出接口

算法模块统一输出事件化 JSON。业务系统只消费该 JSON，不需要理解模型内部细节。

## 1. 通用字段

```json
{
  "module": "fall_risk",
  "device_id": "ezviz_cam_001",
  "person_id": "elder_001",
  "timestamp": "2026-06-26T14:20:00+08:00",
  "scene_region": "bedroom_bedside",
  "risk_level": 3,
  "risk_score": 0.84,
  "confidence": 0.87,
  "trigger_event": "near_fall",
  "risk_factors": [
    "重心快速横向偏移",
    "步态节律中断"
  ],
  "recommended_action": "notify_guardian",
  "evidence_window": {
    "start_time": 15.2,
    "end_time": 17.6
  },
  "model_version": "fall-risk-v0.1"
}
```

## 2. 模块编码

| 编码 | 含义 |
|---|---|
| `fall_risk` | 跌倒风险 |
| `mental_health` | 心理健康风险 |
| `fusion` | 多模块事件汇总 |

## 3. 风险等级

| 等级 | 含义 |
|---:|---|
| 0 | 正常 |
| 1 | 低风险 |
| 2 | 中风险 |
| 3 | 高风险 |
| 4 | 紧急风险 |

## 4. 建议动作

| 编码 | 含义 |
|---|---|
| `record_only` | 仅记录 |
| `observe` | 持续观察 |
| `remind_user` | 建议提醒老人 |
| `notify_guardian` | 建议通知家属 |
| `emergency_alert` | 建议紧急告警 |
| `manual_review` | 建议人工复核 |

## 5. 心理健康模块特别说明

心理健康模块输出的是风险预警，不是医学诊断。高风险输出应进入人工复核或专业人员评估流程，不应由算法直接给出诊断结论。
