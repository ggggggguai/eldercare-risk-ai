# 算法工程架构

更新时间：2026-07-15

## 1. 工程边界

本仓库只实现跌倒风险和心理健康风险的算法原型。两个模块共享人员、设备、时间和事件字段约定，但各自评分并输出自己的 `AlgorithmEvent`。

算法工程负责数据适配、质量控制、特征提取、规则或模型推理、评估工具和标准化事件输出。业务系统负责账号、设备管理、数据库、消息通道、工单和人工处置。

## 2. 代码分层

| 目录 | 职责 |
|---|---|
| `common/` | YAML、JSONL 和 `AlgorithmEvent` 公共契约 |
| `modules/fall_risk/` | 跟踪、姿态、质量控制、步态、坐站、近跌倒、个体基线、跌倒内部评分，以及 manifest、标注校验、split 和事件评估 |
| `modules/mental_health/` | 行为/睡眠适配、日级聚合、个人基线和心理健康评分 |
| `runtime/` | 跌倒实时姿态跟踪、滑动窗口、状态检测和事件节流 |
| `service/` | 跌倒直播流读取、单会话生命周期、FastAPI 和 HTTP 回调 |
| `inference/` | 两个模块的离线命令行入口 |

依赖方向以公共契约和领域模块为内层，运行时和服务为外层。算法模块不访问业务数据库。

## 3. 跌倒风险链路

```text
视频帧
  -> YOLOv8-Pose + ByteTrack
  -> 主目标姿态与质量控制
  -> 10 秒内存窗口
  -> 步态 / 坐站 / 近跌倒 / 跌倒状态 / 个体基线 / 场景特征
  -> 跌倒规则评分卡
  -> fall_risk AlgorithmEvent
  -> 事件节流与后端 HTTP 回调
```

离线 JSONL 脚本仍用于数据处理和调试；实时主路径使用内存对象，不通过临时 JSONL 串联。

与推理链路并行的数据与评估控制链已经具备自动化实现：

```text
本地数据资产
  -> 确定性 manifest + 内容 hash + 媒体元数据
  -> 来源独立的候选标注导入
  -> schema / 来源 / 复核 / 许可严格校验
  -> fall_event / near_fall_event / functional_proxy / longitudinal_baseline 独立 split
  -> 版本化事件匹配、指标与失败案例 bundle
```

该控制链默认失败关闭。当前真实根标签未通过校验，因此四个任务只生成了 `blocked` 产物；合成数据评估只能证明工具链可运行，不能证明真实算法效果。

## 4. 心理健康链路

```text
带绝对时间的姿态记录 + 可选睡眠/自评
  -> 输入适配与质量校验
  -> person_id + 自然日聚合
  -> 3-14 日个人历史基线
  -> 活动、睡眠和规律性偏离评分
  -> 风险等级、置信度和人工复核建议
  -> mental_health AlgorithmEvent
```

当前入口是离线日级 CLI；仓库尚未实现心理健康定时调度或 HTTP 服务。

## 5. 当前实现状态

| 能力 | 状态 | 说明 |
|---|---|---|
| 通用事件契约 | 已实现 | 模块编码只有 `fall_risk` 和 `mental_health` |
| 跌倒离线分层算法 | 已实现 baseline | 各层可独立运行，风险系数尚未用真实老人标签标定 |
| 跌倒实时 HTTP 服务 | 已实现原型 | 单进程单路会话，支持重连、节流和回调 |
| 真实萤石直播联调 | 未完成 | 仍需业务侧直播地址、回调端点和真实设备验证 |
| 心理健康日级闭环 | 已实现 baseline | 支持行为、睡眠、自评、个人基线和缺失模态处理 |
| 心理健康在线调度 | 未实现 | 当前只提供函数和离线 CLI |
| 跌倒数据与评估自动化 | 已实现底座 | 已有统一 manifest、候选标注导入、严格校验、四任务 split builder、事件评估器和合成 bundle |
| 跌倒真实数据门禁 | 阻塞 | 根目录动作/事件标签各 922 条且全部 `pending`，当前严格审计有 2,766 个 schema error；四任务均无合格样本和正式 `split_id` |
| 跌倒正式指标 | 未就绪 | 尚无通过 formal 校验的真实标签、冻结 split、冻结协议和盲测治理证据；合成 bundle 不是比赛结果 |
| 临床有效性 | 未验证 | 所有输出均为风险提示，不是医学诊断 |

## 6. 现行约束

- 第一版优先可解释、可复现和可降级，不把复杂模型作为单点依赖。
- 缺失或低质量数据应降低置信度、限制等级或输出不可用状态，不能填成正常值。
- `person_id` 是跨时间聚合的业务身份；`track_id` 只表示单路视频内轨迹。
- 两个模块分别建立验证集、指标和阈值，不能用一个模块的结果证明另一个模块有效。
