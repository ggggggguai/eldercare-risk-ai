# 算法工程骨架

## 1. 边界

本工程聚焦算法，不做业务系统。两个算法模块分别输出标准化事件 JSON，供后续系统消费。

算法负责：

- 数据读取、清洗、特征提取。
- 模型训练、推理、评估。
- 输出风险等级、风险分数、置信度、风险因子和建议动作。

算法不负责：

- 家属端、社区端、后台页面。
- 消息推送、电话通知、工单流转。
- 账号、权限、设备管理。

## 2. 模块划分

```text
src/elderly_monitoring/common/
  共享 schema、配置读取、JSONL 工具

src/elderly_monitoring/modules/fall_risk/
  跌倒风险算法：姿态、步态、坐站、近跌倒、个体基线、风险融合

src/elderly_monitoring/modules/mental_health/
  心理健康风险算法：活动节律、睡眠节律、社交退缩、日常规律性、问卷/语音文本可选特征

src/elderly_monitoring/modules/fusion/
  多算法事件汇总，只汇总算法输出，不处理业务流程

src/elderly_monitoring/inference/
  单样本、批量、离线特征推理入口
```

## 3. 当前实现策略

第一版不直接上复杂模型，先做可解释轻量主线：

- 跌倒风险：规则评分卡 + 可解释特征，后续接入 Logistic/LightGBM。
- 心理健康：行为节律异常评分 + 人工复核建议，后续再接问卷、语音或文本模型。
- 融合层：只合并两个模块的事件结果，不改变单模块分数含义。
