## 二、P1 待解决问题

| 优先级 | 问题 | 为什么重要 | 建议处理方式 |
|---|---|---|---|
| P1 | 多模态仍偏视觉特征 | 当前主要是视频、姿态、行为节律和场景信息，跨源模态不够强 | 评估是否能加入低成本环境信息，如光照、摄像头区域标签、床边/门口区域状态 |
| P1 | 个体化基线冷启动不足 | 新用户前 3-7 天没有稳定个人基线，容易影响早期判断 | 增加“无基线阶段”的群体阈值策略和基线逐步更新策略 |
| P1 | 误报控制策略还需细化 | 居家场景误报会影响可用性和评审印象 | 增加连续触发抑制、低置信度人工复核、短时间重复事件合并 |
| P1 | 近跌倒阈值需要校准 | 近跌倒是核心创新点，但也最容易误报 | 用盲测集统计阈值变化对召回率和误报率的影响 |
| P1 | 老年人差异覆盖不足 | 不同身体状态会影响步态、起身、转身特征 | 数据采集时记录年龄段、行动能力、是否使用拐杖、是否慢病等基础信息 |
| P1 | 场景鲁棒性测试矩阵不够完整 | 实际居家场景有低光、遮挡、多人、家具遮挡等情况 | 建立场景矩阵：白天/夜间、正面/侧面、遮挡/无遮挡、单人/多人 |
| P1 | 提前预警案例不足 | 前置预警需要可视化案例支撑 | 准备 2-3 个典型时间轴案例，展示风险从 1 级升到 3 级的过程 |
| P1 | 算法与总系统接口需联调 | 算法输出如果无法被总系统消费，会影响落地表达 | 提供 mock JSON 和示例推理结果给开发部门联调 |

## 四、建议下一步顺序

1. 先确认萤石平台输入与算法 JSON 输出字段。
2. 同步准备隐私授权模板和真实/半真实数据采集清单。
3. 先实现轻量主线模型，跑通从视频到风险等级输出的闭环。
4. 建立盲测集并统计第一版指标。
5. 再决定是否加入 ST-GCN、Transformer AutoEncoder 或 Risk Fusion Transformer 增强实验。
